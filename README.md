# Stalker 2D

Stalker 2D — это пошаговая клиент-серверная 2D-игра в духе сталкерской тематики с упором не на сложную графику, а на геймплей, архитектуру и качество кода и применяемые технологии.

## Идея проекта

Проект создаётся как основа для игры с простой 2D-картой, исследованием мира, взаимодействием с окружением и персонажами, а в будущем — с поддержкой мультиплеера и пошагового режима.

Главная цель — сделать не просто “игру, которая работает”, а понятный, расширяемый и аккуратно спроектированный кодовый проект.

## Основные цели

- реализовать базовую клиент-серверную архитектуру;
- построить понятную игровую модель мира;
- сделать основу для дальнейшего расширения механик;
- сохранить код чистым, модульным и поддерживаемым;
- подготовить проект к будущему мультиплееру;
- использовать проект как площадку для практики C++, сетевого взаимодействия и проектирования.

## Что планируется в результате

На первом этапе проект должен позволять:

- запускать сервер и клиент отдельно;
- подключать клиента к серверу;
- отображать простую 2D-карту;
- перемещать игрока по миру;
- обрабатывать базовые игровые действия;
- передавать состояние между сервером и клиентом;
- иметь основу для добавления NPC, диалогов, предметов и боевой системы.

## Приоритеты проекта

В проекте приоритет отдается:

1. качеству архитектуры;
2. понятному разделению ответственности между компонентами;
3. корректной клиент-серверной модели;
4. расширяемости;
5. только потом — визуальной сложности и эффектам.

## Техническая направленность

Проект разрабатывается на C++ и строится вокруг клиент-серверного взаимодействия.

Планируемые технические принципы:

- сервер является источником истины для игрового состояния;
- клиент отвечает за отображение и отправку пользовательских действий;
- сетевое взаимодействие строится на Boost.Asio + TCP;
- сериализация сообщений — через Protocol Buffers;
- архитектура проектируется с заделом на будущий мультиплеер.

## Текущий фокус разработки

Сейчас основной фокус проекта:

- заложить базовую структуру проекта;
- настроить сборку и зависимости;
- определить формат сетевого взаимодействия;
- реализовать минимальный игровой цикл;
- создать фундамент для дальнейших игровых механик.

## Планы на будущее

В дальнейшем в проект могут быть добавлены:

- NPC и система диалогов;
- инвентарь и предметы;
- квесты;
- взаимодействие с объектами мира;
- пошаговый мультиплеер;
- синхронизация нескольких игроков;
- сохранение состояния;
- более сложная логика мира.
